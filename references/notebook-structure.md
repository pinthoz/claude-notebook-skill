# Notebook Structure (Academic and Complete)

Use this as the default structure for every notebook generated by this skill.

## 1) Title
- Specific, research oriented, and aligned with the target task.

## 2) Introduction
- Context, motivation, and expected impact.
- Domain background needed to interpret results.
- Scope of analysis and exclusions.

## 2.1) Index
- Add a notebook index after the introduction.
- Include links to major sections for fast navigation.

## 3) Problem framing and objectives
- Task type and target definition.
- Main objective and secondary objectives.
- Formal success criteria and operational constraints.
- Hypotheses to test.

## 4) Data provenance and governance
- Data sources, acquisition method, and snapshot date.
- Data usage assumptions and ethical notes when relevant.
- Potential sampling biases and representativeness caveats.

## 5) Data overview
- Dataset dimensions and feature taxonomy.
- Data types and semantic meaning of key variables.
- Target distribution and imbalance status.
- Dataset preview before cleaning.

## 6) Data quality audit
- Missing values and missingness mechanisms.
- Duplicates, inconsistent records, and data validation checks.
- Outlier diagnostics with rationale for treatment.
- Leakage audit, including temporal and proxy leakage.
- Dataset preview after cleaning.

## 7) Exploratory data analysis (EDA)
- Univariate exploration with robust descriptive statistics.
- Bivariate and multivariate analysis against target.
- Correlation and dependency analysis.
- Segment level distributions for relevant cohorts.
- Short interpretation after each major figure group.

## 8) Data splitting strategy
- Holdout strategy and rationale.
- Cross validation design aligned with data structure.
- Stratification, grouping, or temporal split rules.

## 9) Preprocessing and feature engineering
- Leakage safe preprocessing pipeline.
- Imputation, encoding, scaling, and transformations.
- Feature construction and feature selection rationale.
- Notes on computational cost and reproducibility.

## 9.1) Representation and geometry diagnostics
- Add dimensionality reduction visual analysis.
- Include PCA projections.
- Include t-SNE projections where computationally feasible.

## 10) Baseline and candidate models
- Transparent baseline with expected limitations.
- Diverse model families, not a single algorithm family.
- Model assumptions and expected strengths.

## 11) Hyperparameter optimization
- Search space design and justification.
- Search method and computational budget.
- Model selection criteria and tie breaking policy.
- Include GridSearchCV or HalvingRandomSearchCV when appropriate.

## 11.1) Validation variants
- Use StratifiedKFold for classification by default.
- Use StratifiedGroupKFold when grouped leakage is possible.
- Use GroupShuffleSplit for grouped holdout evaluation where needed.
- Use cross_val_predict for out-of-fold diagnostics and calibration analysis.

## 12) Evaluation metrics
- Multi metric framework by task type.
- Classification: accuracy, balanced accuracy, precision, recall, F1, ROC AUC, PR AUC, log loss, calibration.
- Regression: MAE, MSE, RMSE, R2, adjusted R2 where applicable, MAPE or sMAPE where meaningful.
- Report both central tendency and variability across folds.

## 13) Cross validation results
- Fold level table for each model.
- Aggregate table with mean and dispersion.
- Ranking with metric trade off commentary.

## 14) Final holdout evaluation
- Evaluate only selected candidates on holdout.
- Include confidence discussion and practical decision implications.

## 15) Interpretability and feature importance
- Global importance methods, including permutation importance.
- Local explanations for representative and difficult cases.
- Plausibility checks and potential leakage warnings.

## 16) Error analysis
- Confusion and class wise errors or residual diagnostics.
- Segment wise underperformance patterns.
- Concrete hypotheses for improvement.

## 17) Robustness and sensitivity analysis
- Stability across random seeds.
- Sensitivity to thresholds for classification decisions.
- Ablation studies for critical features and preprocessing steps.

## 18) Limitations and threats to validity
- Internal validity limits.
- External validity limits.
- Data and modeling assumptions that may fail.

## 19) Reproducibility appendix
- Environment and package versions.
- Seeds, deterministic settings, and execution notes.
- Runtime and hardware notes for expensive experiments.

## 19.1) Model artifact
- Save the final selected model to `models/` as a `.pkl` file.
- Create `models/` automatically when it does not exist.

## 20) Conclusion (to be written)
- Keep only prompts for future writing.
- Do not generate the final narrative conclusion text.
